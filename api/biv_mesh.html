BivMesh

Biventricular mesh class for cardiac modeling.

class biv_lite.biv_mesh.BivMesh(control_points: ndarray, name: str = 'biv_mesh', model_folder: Path = PosixPath('/home/runner/work/biv-lite/biv-lite/src/biv_lite/model'))[source]

Bases: Mesh

Handle biventricular cardiac mesh models.

This class extends the Mesh base class to facilitate the construction, manipulation, and analysis of biventricular meshes. It provides utilities to load fitted models, manipulate control points, and extract specific components (LV/RV endocardial and epicardial surfaces), with support for template models, subdivision matrices, and component re-indexing.

to_obj(output_filename: Path, components: List[IntEnum] = None)[source]

Export the BiVMesh object to a Wavefront OBJ file format. This method writes the mesh geometry and material assignments to a .obj file, organizing faces by component/material groups.

Parameters:

  • output_filename (Path) – Path to the output OBJ file.

  • components (List[IntEnum], optional) – List of component enums to include in the export. If None, all available components are included.

Returns:

None

Raises:

IOError – If the output file cannot be written.

Note

Vertex indices are automatically converted to 1-based indexing as required by the OBJ format specification.

Note

Components with no associated faces are skipped in the output.

Example:
>>> mesh = BiVMesh(...)
>>> mesh.to_obj("biv.obj")
>>> mesh.to_obj("rvlv.obj", components=[Components.LV_ENDOCARDIAL, Components.RV_FREEWALL, Components.RV_SEPTUM])
